# EIP-7702 Support for morph-skills

**Date:** 2026-03-31
**Status:** Approved
**Reference implementation:** morph-l2/morph-cli (`src/lib/chain/eip7702.ts`)

---

## Overview

Add EIP-7702 (EOA delegation, tx type `0x04`) support to `morph-skills`, fully aligned with the existing `morph-cli` implementation. Five new commands mirror the `altfee-*` command group pattern. Local private-key signing only; Social Login Wallet users are routed to BGW.

---

## Architecture

```
scripts/
  morph_api.py          # entry point + existing code unchanged (~1957 lines)
  morph_7702.py         # new: EIP-7702 implementation (~300 lines)
                        #   imports helpers from morph_api (rpc_call, rlp_encode, etc.)
                        #   exports register_7702_commands(subparsers) for argparse

skills/morph-7702/SKILL.md   (new sub-skill)
SKILL.md                     (root, add morph-7702 section + command table entries)
```

`morph_7702.py` imports shared helpers from `morph_api.py` (`rpc_call`, `rlp_encode`, `_ok`, `_err`, etc.) at the top level. `morph_api.py` imports `morph_7702` **inside `build_parser()`** (late import, not top-level) to avoid circular imports, then calls `register_7702_commands(subparsers)`. Agent-facing CLI interface is unchanged: `python3 scripts/morph_api.py 7702-batch ...`.

## Commands

### `7702-delegate`

Check whether an EOA has been delegated via EIP-7702.

```bash
python3 scripts/morph_api.py 7702-delegate --address 0xEOA
```

Output:
```json
{
  "success": true,
  "data": {
    "address": "0xEOA",
    "delegated": true,
    "contract": "0xBD7093Ded667289F9808Fa0C678F81dbB4d2eEb7",
    "code_prefix": "0xef0100"
  }
}
```

Implementation: `eth_getCode(address)` → check if starts with `0xef0100`. No private key required.

---

### `7702-authorize`

Sign a 7702 authorization object offline. Returns the signed authorization — no transaction is sent. For wallets that want to collect the authorization and bundle it into a transaction later.

```bash
python3 scripts/morph_api.py 7702-authorize \
  --private-key 0xKey \
  [--delegate 0xContract]   # default: SimpleDelegation
```

Output:
```json
{
  "success": true,
  "data": {
    "chainId": 2818,
    "contractAddress": "0xBD7093Ded667289F9808Fa0C678F81dbB4d2eEb7",
    "nonce": 5,
    "yParity": "0x01",
    "r": "0x...",
    "s": "0x..."
  }
}
```

---

### `7702-send`

Send a single call using EIP-7702 delegation. The EOA temporarily acts as a smart contract for this one transaction.

```bash
python3 scripts/morph_api.py 7702-send \
  --to 0xContract \
  --value 0.01 \
  --data 0xCalldata \
  --private-key 0xKey \
  [--delegate 0xContract]   # default: SimpleDelegation
  [--gas 200000]
```

`--value` defaults to `0` (no ETH transfer). `--data` defaults to `0x` (empty calldata). At least one of `--value` or `--data` must be provided.

Output:
```json
{ "success": true, "data": { "tx_hash": "0x..." } }
```

---

### `7702-batch`

Atomically execute multiple calls via SimpleDelegation contract. This is the primary use case: Agent can approve + swap + transfer in a single atomic transaction.

```bash
python3 scripts/morph_api.py 7702-batch \
  --calls '[{"to":"0xAddr","value":"0","data":"0xCalldata"},{"to":"0xAddr2","value":"0.01","data":"0x"}]' \
  --private-key 0xKey \
  [--delegate 0xContract]   # default: SimpleDelegation
  [--gas 300000]
```

`--calls` is a JSON array of `{to, value, data}` objects. `value` is in ETH (human-readable). `data` defaults to `"0x"` (empty calldata) if omitted.

Output (broadcast):
```json
{ "success": true, "data": { "tx_hash": "0x...", "calls_count": 2 } }
```

---

### `7702-revoke`

Revoke the 7702 delegation by sending an authorization with `address(0)` as the delegate.

```bash
python3 scripts/morph_api.py 7702-revoke \
  --private-key 0xKey
```

## Core Technical Implementation

### Constants

```python
SIMPLE_DELEGATION    = "0xBD7093Ded667289F9808Fa0C678F81dbB4d2eEb7"
DELEGATION_PREFIX    = "0xef0100"   # on-chain code prefix for delegated EOAs
EIP7702_TYPE_BYTE    = 0x04         # tx envelope type
AUTH_MAGIC_BYTE      = 0x05         # authorization hash domain separator
REVOKE_ADDR          = "0x0000000000000000000000000000000000000000"
GAS_FALLBACK_SEND    = 200_000      # fallback if estimateGas fails (single call)
GAS_FALLBACK_BATCH   = 300_000      # fallback if estimateGas fails (batch)
GAS_REVOKE           = 80_000       # fixed gas for revoke
```

### Authorization Hash

```python
# keccak256(0x05 || RLP([chainId, contractAddress, nonce]))
def compute_auth_hash(chain_id: int, contract_addr: str, nonce: int) -> bytes:
    rlp_encoded = rlp_encode([
        rlp_int(chain_id),
        bytes.fromhex(contract_addr[2:]),
        rlp_int(nonce),
    ])
    return keccak256(bytes([AUTH_MAGIC_BYTE]) + rlp_encoded)
```

Uses the existing `rlp_encode` / `rlp_int` helpers already in `morph_api.py` (the altfee section).

### Geth Nonce Offset (critical)

Geth-based chains (including Morph) increment the sender nonce **before** processing the authorization list. For self-delegation (sender == authority, which is the standard pattern):

```python
tx_nonce   = get_transaction_count(eoa_address)   # tx uses this nonce
auth_nonce = tx_nonce + 1                          # auth must use tx_nonce + 1
```

This is transparent to the user — the command computes it automatically.

### Type 0x04 Transaction Serialization

```
0x04 || RLP([
  chainId,
  nonce,               # tx_nonce
  maxPriorityFeePerGas,  # 0 (L2)
  maxFeePerGas,
  gas,
  to,
  value,
  data,
  [],                  # accessList (always empty)
  authorizationList,   # [[chainId, contractAddr, nonce, yParity, r, s], ...]
  yParity,             # tx signature
  r,
  s,
])
```

Extends the existing `_sign_altfee_tx` pattern in `morph_api.py` — same manual RLP approach, new field layout.

### 7702-batch Signing Flow (8 steps)

```
1. eth_getTransactionCount(eoa)             → tx_nonce
2. eth_call SimpleDelegation.nonce(eoa)     → delegation_nonce
3. abi.encode(calls, delegation_nonce, chain_id, eoa) → keccak256 → data_hash
4. eth_account.sign_message(data_hash)      → EIP-191 signature (auto-prefixes)
5. abi.encode execute(calls, delegation_nonce, sig) → calldata
6. compute_auth_hash(chain_id, SIMPLE_DELEGATION, tx_nonce+1) → sign → auth_sig
7. serialize_eip7702_tx(to=eoa, data=calldata, auth_list=[auth_sig]) → raw_tx
8. eth_sendRawTransaction(raw_tx)           → tx_hash
```

### Python Dependencies

No new libraries required. All needed is already in the existing dependency set:
- `eth_account` — `sign_message()` for EIP-191 data_hash signing
- `eth_keys` — `PrivateKey.sign_msg_hash()` for auth hash and type 0x04 tx signing
- `eth_abi` — `encode()` for batch call ABI encoding
- `eth_utils` / `eth_hash` — keccak256
- `requests` — RPC calls

The existing `rlp_encode` + `rlp_int` helpers in the altfee section are reused for auth hash and tx serialization.

## Error Handling

| Scenario | Behavior |
|---|---|
| `--calls` is not valid JSON | `{"success": false, "error": "invalid --calls JSON: <msg>"}` |
| `--calls` entry missing required field | `{"success": false, "error": "each call must have 'to', 'value', 'data'"}` |
| `eth_estimateGas` fails | fallback to `GAS_FALLBACK_SEND` or `GAS_FALLBACK_BATCH`, include `"gas_estimate": "fallback"` in output |
| EOA already delegated on `7702-batch` | works normally; authorization list overrides the existing delegation for this tx |
| altfee + 7702-send used at the same time | not possible — they are separate commands; documented as mutually exclusive in SKILL.md |
| Private key invalid | existing `_load_account` handles this before any RPC call |

## Out of Scope

- Social Login Wallet / BGW TEE signing for 7702 transactions (BGW routing documented in SKILL.md)
- EIP-7702 support inside existing commands via `--eip7702` flag (deferred; separate commands are cleaner)
- x402 payment protocol (separate spec)
- Testnet (Hoodi) support (can be added later via `--hoodi` flag following altfee pattern)
